# Non-24-hour-cosinor-analysis
Utilities for cosinor analysis of irregularly sampled time series with an unknown period. The model estimates period, amplitude, phase, and MESOR using a combination of grid search, optimization, and linear regression, without assuming a fixed cycle such as 24 hours.

This script provides a workflow for simulating and analyzing oscillatory time series using a cosinor model with an unknown period. First, synthetic data can be generated by calling the simulation function with parameters such as the number of observations, true period, amplitude, phase shift, baseline level, noise level, and maximum observation time. The function returns a data frame containing irregularly spaced time points, noisy observations, and the corresponding noise-free signal.

To perform analysis, the user extracts the time and observation vectors from the data and passes them to the fitting function. Optional arguments allow specification of the search range for the period, the resolution of the initial grid search, and the tolerance for the refinement step. The fitting procedure automatically removes invalid values, conducts a coarse grid search to identify candidate periods, and then refines the estimate using numerical optimization. For each candidate period, the remaining parameters are estimated efficiently using linear regression.

The resulting object contains the estimated period, amplitude, phase, and mean level, along with fitted values, residuals, and goodness-of-fit statistics. These elements can be accessed directly by name from the returned list for further analysis or visualization. A separate summary function can then be applied to the fitted result to display a structured and interpretable report, including the estimated peak timing within one cycle. This workflow supports unevenly sampled data and does not require assuming a predefined periodicity such as a 24-hour rhythm.


## Usage

This script provides a workflow for simulating and analyzing oscillatory time series using a cosinor model with an unknown period.

### Model

The cosinor model is defined as:

$$
y = A \cdot \cos\left(\frac{2\pi x}{T} + \phi\right) + C + \varepsilon
$$

where

* $T$: period (estimated)
* $A$: amplitude
* $\phi$: phase shift
* $C$: MESOR (mean level)
* $\varepsilon$: Gaussian noise


### 1. Generate Sample Data: `generate_timeseries`

Synthetic irregularly sampled time series can be generated by specifying parameters such as the number of observations, true period, amplitude, phase, baseline, and noise level.

The function returns a data frame with:

* $x$: observation times
* $y$: noisy observations
* $y_{\text{true}}$: noise-free signal


### 2. Fit the Cosinor Model: `fit_cosinor_free_period`

Provide the observation times and values to the fitting procedure. The algorithm estimates the unknown period and other parameters.

The estimation consists of two steps:

1. **Coarse grid search** over a range $T \in [T_{\min}, T_{\max}]$
2. **Refined optimization** around the best candidate

For a fixed period $T_0$, the model is rewritten as:

$$
y = C + \beta_1 \cos\left(\frac{2\pi x}{T_0}\right) + \beta_2 \sin\left(\frac{2\pi x}{T_0}\right)
$$

and parameters are estimated using ordinary least squares.

The amplitude and phase are recovered as:

$$
A = \sqrt{\beta_1^2 + \beta_2^2}
$$

$$
\phi = {\rm atan2}(-\beta_2, \beta_1)
$$


### 3. Output and Interpretation: `print_cosinor_summary`

The fitted result includes:

* Estimated period $T$
* Amplitude $A$
* Phase $\phi$
* MESOR $C$
* Fitted values and residuals
* Residual sum of squares (RSS)
* Coefficient of determination R$^2$

The peak timing within one cycle is given by:

$$
x_{\text{peak}} = \left( -\frac{\phi}{2\pi} \cdot T \right) \bmod T
$$

### 4. Example Workflow

To run a complete example of the analysis, first load the functions defined in
`cosinor_analysis.R` and then execute the demonstration script `cosinor_example.R`.
.

After loading the main script, the example script performs the full workflow automatically. It generates an irregularly sampled time series, fits the cosinor model with a free period, visualizes the observed data together with the fitted curve, and prints a summary of the estimated parameters.

This example allows users to confirm that the implementation is working correctly and to understand how the estimation behaves when the true period deviates from a canonical value such as 24 hours.

By examining the output, users can verify that the estimated period, amplitude, phase, and MESOR are close to the true values used in the simulation, and can also visually assess the quality of the fit through the plotted results.
