# Non-24-hour-cosinor-analysis
Utilities for cosinor analysis of irregularly sampled time series with an unknown period. The model estimates period, amplitude, phase, and MESOR using a combination of grid search, optimization, and linear regression, without assuming a fixed cycle such as 24 hours.

This script provides a workflow for simulating and analyzing oscillatory time series using a cosinor model with an unknown period. First, synthetic data can be generated by calling the simulation function with parameters such as the number of observations, true period, amplitude, phase shift, baseline level, noise level, and maximum observation time. The function returns a data frame containing irregularly spaced time points, noisy observations, and the corresponding noise-free signal.

To perform analysis, the user extracts the time and observation vectors from the data and passes them to the fitting function. Optional arguments allow specification of the search range for the period, the resolution of the initial grid search, and the tolerance for the refinement step. The fitting procedure automatically removes invalid values, conducts a coarse grid search to identify candidate periods, and then refines the estimate using numerical optimization. For each candidate period, the remaining parameters are estimated efficiently using linear regression.

The resulting object contains the estimated period, amplitude, phase, and mean level, along with fitted values, residuals, and goodness-of-fit statistics. These elements can be accessed directly by name from the returned list for further analysis or visualization. A separate summary function can then be applied to the fitted result to display a structured and interpretable report, including the estimated peak timing within one cycle. This workflow supports unevenly sampled data and does not require assuming a predefined periodicity such as a 24-hour rhythm.
